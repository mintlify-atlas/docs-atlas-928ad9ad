---
title: "Quickstart: run TUIC server and client in minutes"
description: "Install tuic-server and tuic-client, write a minimal config for each, and verify your SOCKS5 proxy is tunneling traffic over QUIC in minutes."
---

This guide walks you through running a TUIC server and connecting to it with the TUIC client. By the end you will have a working SOCKS5 proxy at `127.0.0.1:1080` tunneled over an encrypted QUIC connection.

<Steps>

<Step title="Install the server">

Install `tuic-server` using Cargo or download a prebuilt binary from [GitHub Releases](https://github.com/Itsusinn/tuic/releases).

<CodeGroup>

```bash cargo
cargo install --git https://github.com/Itsusinn/tuic.git tuic-server
```

```bash binary (Linux x86_64)
curl -Lo tuic-server https://github.com/Itsusinn/tuic/releases/latest/download/tuic-server-x86_64-unknown-linux-gnu
chmod +x tuic-server
sudo mv tuic-server /usr/local/bin/
```

</CodeGroup>

</Step>

<Step title="Generate a starter config">

Run `tuic-server --init` to write an example configuration file in the current directory, then open it for editing.

```bash
tuic-server --init
```

This creates a `config.toml` file with all available options commented and explained.

</Step>

<Step title="Write a minimal server config">

Replace the generated file with this minimal working configuration. Fill in your own values for `uuid`, `password`, `certificate`, and `private_key`.

```toml config.toml
log_level = "info"

# Listen on all interfaces, port 443
server = "[::]:443"

zero_rtt_handshake = false
auth_timeout = "3s"
task_negotiation_timeout = "3s"
gc_interval = "10s"
gc_lifetime = "30s"
max_external_packet_size = 1500
stream_timeout = "60s"

[users]
# Replace with your own UUID and a strong password
f0e12827-fe60-458c-8269-a05ccb0ff8da = "your_strong_password"

[tls]
# Path to your TLS certificate and private key
certificate = "/path/to/cert.crt"
private_key = "/path/to/key.crt"
hostname = "your.domain.com"

[outbound.default]
type = "direct"
ip_mode = "v4first"
```

<Note>
  If you do not have a TLS certificate yet, set `self_sign = true` and `hostname = "localhost"` in `[tls]` to use an auto-generated self-signed certificate. On the client side you will need to set `skip_cert_verify = true`.
</Note>

<Tip>
  To use automatic Let's Encrypt certificates instead of providing your own, set `auto_ssl = true` and `hostname = "your.domain.com"` in `[tls]`. The server must be reachable on port 80 for the ACME HTTP-01 challenge.
</Tip>

</Step>

<Step title="Start the server">

Pass the config file path with `-c`:

```bash
tuic-server -c config.toml
```

You should see log output similar to:

```text
INFO tuic_server: Listening on [::]:443
```

<Warning>
  Port 443 requires root or the `CAP_NET_BIND_SERVICE` capability on Linux. To grant it without running as root: `sudo setcap CAP_NET_BIND_SERVICE=+eip $(which tuic-server)`
</Warning>

</Step>

<Step title="Install the client">

Install `tuic-client` using Cargo or download a prebuilt binary from [GitHub Releases](https://github.com/Itsusinn/tuic/releases).

<CodeGroup>

```bash cargo
cargo install --git https://github.com/Itsusinn/tuic.git tuic-client
```

```bash binary (Linux x86_64)
curl -Lo tuic-client https://github.com/Itsusinn/tuic/releases/latest/download/tuic-client-x86_64-unknown-linux-gnu
chmod +x tuic-client
sudo mv tuic-client /usr/local/bin/
```

</CodeGroup>

</Step>

<Step title="Write a minimal client config">

Create a `client.toml` with your server address, the UUID and password you set on the server, and a local SOCKS5 listen address.

```toml client.toml
log_level = "info"

[relay]
# Your server's hostname or IP address and port
server = "your.domain.com:443"

# Must match the UUID in the server's [users] table
uuid = "f0e12827-fe60-458c-8269-a05ccb0ff8da"

# Must match the password for that UUID on the server
password = "your_strong_password"

udp_relay_mode = "native"
congestion_control = "cubic"
zero_rtt_handshake = false
timeout = "8s"
heartbeat = "3s"

[local]
# SOCKS5 proxy will listen here
server = "127.0.0.1:1080"
dual_stack = true
max_packet_size = 1500
```

<Info>
  If you used a self-signed certificate on the server, add `skip_cert_verify = true` under `[relay]`. This is insecure — only use it for local testing.
</Info>

</Step>

<Step title="Start the client">

```bash
tuic-client -c client.toml
```

You should see log output confirming the SOCKS5 server is listening:

```text
INFO tuic_client: SOCKS5 server listening on 127.0.0.1:1080
```

</Step>

<Step title="Test the proxy">

Send a request through the SOCKS5 proxy to verify traffic is being relayed:

```bash
curl --proxy socks5h://127.0.0.1:1080 https://ipinfo.io/ip
```

If the request succeeds and returns your server's IP address, the proxy is working correctly.

</Step>

</Steps>
