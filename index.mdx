TUIC is a delicately-tuned 0-RTT proxy protocol designed to minimize handshake latency while keeping the protocol simple and efficient. Built on top of QUIC, it achieves zero round-trip TCP and UDP proxying with full multiplexing, connection migration, and bidirectional congestion control — all without the overhead of traditional proxy protocols.

## Why TUIC?

TUIC is engineered for minimal latency and maximum throughput. By building on QUIC, it avoids the TCP head-of-line blocking problem and delivers native 0-RTT connection establishment for both TCP and UDP traffic.

## Get started

<Steps>
  <Step title="Install the server">
    Download a prebuilt binary or install via Cargo, then generate an example config with `tuic-server --init`.
  </Step>
  <Step title="Configure TLS and users">
    Set your hostname, enable ACME or provide certificates, and add user UUIDs with passwords to `[users]`.
  </Step>
  <Step title="Start the server">
    Run `tuic-server -c config.toml` and verify it is listening on your configured port.
  </Step>
  <Step title="Connect with the client">
    Configure `tuic-client` with your server address, UUID, and password. Start it and point your applications at `127.0.0.1:1080` (SOCKS5).
  </Step>
</Steps>

<Note>
  TUIC is a fork of the original [tuic-protocol/tuic](https://github.com/tuic-protocol/tuic) with significant enhancements including ACME support, ACL rules, RESTful API, BBR3 congestion control, and Docker image builds.
</Note>
